# ModPack 格式

除标准的 `.mod.zip` 压缩包外，ModLoader 还支持 `.modpack` 二进制格式。`.modpack` 是 ModLoader 自定义的二进制 Mod 包格式，适用于对体积或加载性能有更高要求的场景。

## 与 .mod.zip 的区别

| 特性     | .mod.zip                 | .modpack               |
| -------- | ------------------------ | ---------------------- |
| 格式     | 标准 Zip 压缩            | 自定义二进制           |
| 可读性   | 可直接用压缩工具解压查看 | 二进制格式，需专用工具 |
| 体积     | 取决于 Zip 压缩率        | 可针对 Mod 场景优化    |
| 打包方式 | 手动打包或 packModZip.js | ModLoader 工具链       |

## 适用场景

- **体积优化**：某些场景下可能获得更小的文件体积
- **加载性能**：二进制解析可能比 Zip 解压更快（取决于实现）
- **工具链集成**：作为 ModLoader 构建流程的中间产物

:::info
对于大多数 Mod 作者，推荐使用 `.mod.zip` 格式。其标准、通用，且可通过任意 Zip 工具手动打包，无需额外工具。
:::

## 生成方式

`.modpack` 通常由 ModLoader 的打包工具或构建流程生成，而非手动创建。具体生成方式请参考 ModLoader 源码中的相关实现。

若你使用 `.mod.zip` 打包 Mod，ModLoader 会正常识别并加载，无需转换为 `.modpack`。
